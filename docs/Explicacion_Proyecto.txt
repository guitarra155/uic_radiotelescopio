=============================================================================
CONTEXTO GLOBAL DEL PROYECTO: UIC RADIOTELESCOPIO
=============================================================================

El proyecto "UIC Radiotelescopio" es una plataforma de software de escritorio orientada al Procesamiento Digital de Señales (DSP) para un radiotelescopio sintonizado en la frecuencia o línea de emisión de Hidrógeno Neutro (HI) a 1420.40 MHz.

La aplicación permite adquirir, procesar y visualizar señales de radiofrecuencia (RF) ya sea en tiempo real (mediante hardware SDR "Software-Defined Radio" como RTL-SDR o HackRF), o bien mediante la reproducción de archivos de grabación de señales locales en formato I/Q (.iq). 

Tecnologías principales:
• Interfaz Gráfica (UI): Construida en Python usando el framewok "Flet" (basado indirectamente en Flutter) lo que otorga una experiencia gráfica moderna, atractiva y con un tema oscuro "Dark Mode".
• Procesamiento de Señales: Numpy.
• Visualización/Gráficos: Matplotlib (Corriendo en segundo plano "Agg" y renderizado en imágenes en Base64 sobre la UI de Flet).

=============================================================================
¿CÓMO FUNCIONA? ARCHITECTURE OVERVIEW 
=============================================================================

El sistema está diseñado en una arquitectura de 2 capas principales y comunicación asíncrona:

1. El Motor de Procesamiento (DSP Engine): Ejecutándose en un hilo separado o "Worker Thread", lee los datos brutos (muestras I/Q) del archivo o del hardware sin bloquear (congelar) la interfaz de usuario. Transforma esta información (cálculo de Amplitud e implementación de FFT - Transformada Rápida de Fourier para el espectro), rellenando buffers circulares internos con los arreglos matemáticos listos.
   
2. La Interfaz Gráfica (Flet + Charts): La UI posee un bucle asíncrono "refresh_loop" corriendo ~60 frames por segundo. Constantemente le pide al generador de gráficos en Matplotlib que pinte los datos guardados en el "motor". Estos gráficos convertidos a imagen PNG (Base64) se actualizan en pantalla simulando un stream en tiempo real.

=============================================================================
EXPLICACIÓN DEL CÓDIGO (ARCHIVO POR ARCHIVO)
=============================================================================

1. main.py (Punto de entrada)
   - Configura la ventana inicial de Flet y aplica la paleta de colores.
   - Instancia todos los componentes de diseño visual (Header, Footer, Barra de Pestañas).
   - Inicia un bucle permanente "async def refresh_loop()" que avisa a las pestañas a través del bus de mensajería (pubsub) cuando el stream de datos está activo, para que ellas rendericen y actualicen de nuevo los gráficos.
   - Lanza la aplicación.

2. dsp_engine.py (El Cerebro)
   - Contiene la clase fundamental "DSPEngine" tratada como "Singleton" (una solita instancia global "engine_instance").
   - Posee las variables de configuración del hardware y variables de control (Db_min, Db_max, Frecuencia_min, Frecuencia_max, cascada de historia).
   - `start_stream()`: Lanza un hilo (Thread) que comienza a buscar datos.
   - `_process_file_loop()`: Lee fragmentos de un archivo `.iq` local de acuerdo a su formato puro binario ("uint8", "int8", etc.), transformando bytes a componentes complexos "IQ".
   - `_process_dsp_core()`: La parte matemática dura. Agarra la señal cruda "iq". Toma tramos para crear la "Amplitud", ejecuta una ventana de Hanning y un promediado en "batches" (lotes) de la FFT para generar un Espectro estable, sube esta información de nuevo al "Waterfall" (el gráfico histórico de calor) y extrae muestras aleatorias simulando un "Histograma". 
   - Notar que el streaming SDR `_process_sdr_loop()` incluye una simulación o "cielo virtual" inyectando ruido mediante NumPy si no hay equipos conectados.

3. charts.py (Renderizador de Gráficos Matplotlib)
   - Utiliza Matplotlib en backend "Agg" (previene que Matplotlib intente abrir ventanas externas).
   - Extrae la información limpia desde `engine_instance`.
   - `chart_amplitude()`: Dibuja la curva de potencia bruta en el tiempo contra la amplitud midiendo su voltaje.
   - `chart_spectrum()`: Plotea el espectro de frecuencias usando la información de la FFT. Marca claramente 1420.40 MHz con una barrera ámbar.
   - `chart_spectrogram()`: Plotea un gráfico mapa de calor (Waterfall) en memoria 2D hacia un espacio imagen, usando "imshow" (de colores "inferno").
   - `chart_histogram()`: Plotea un histograma con la dispersión de error y aproxima una curva normal o campana de Gauss ideal encima de los datos adquiridos.
   - Todos terminan con `fig_to_b64()` que condensa los bits de las gráficas listas a un texto "data:image/png;base64".

4. constants.py
   - Define códigos hexadecimales de colores, fondos o decorativos para la interfaz gráfica y para Matplotlib.

5. Directorio "components" (Componentes visuales estáticos o UI Genérica)
   - `shared.py`: Funciones auxiliares que evitan escribir atributos repetitivos como los "borders", "TextFields", y los paneles con radio de Flet. Plegables y fáciles de reutilizar.
   - `layout.py`: Programa la Cabecera/Header que tiene información visual, y sobre todo un botón dinámico de Iniciar/Detener streaming y un botón rojo global de "Emergencia Parada". También configura el final inferior de página `build_footer()`.

6. Directorio "tabs" (Lógica independiente por cada Pestaña del App)
   Todas las pestañas exportan una función "build_NOMBRE" y están suscritas activamente actualizando su gráfica interna si llega el mensaje "refresh_charts":
   
   - `monitoring.py`: Interfaz principal. Une el visor local de Amplitud (arriba) y el visor de Espectro (abajo) junto a un panel lateral simulando un algoritmo detector avanzado e interruptores de "RFI" falso con reportes duros.
   - `spectrogram.py`: Construye en full width el gráfico de Spectrogama map-color y adjunta una bella cajita con la leyenda visual abajo.
   - `statistics.py`: El visualizador del histograma y el simulado trigger anomalo Smart. Incluye entrada texto para cambiar dinámicamente el "Threshold" de detección anómalo (Umbral%).
   - `sdr_config.py`: Panel del usuario. Conecta las entradas de la UI usando selectores, checkboxes o cajones de texto (rutas de los ".iq", formato, limites DBFS Y y Zoom Frecuencia X) inyectando o mutando directamente la `engine_instance` base. Contiene el tutorial estático y detalles de máquina.
